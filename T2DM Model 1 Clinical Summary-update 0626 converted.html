Cohort and Target Definition

Cohort & Target Definition

The model was built on 299,224 empaneled T2DM patients from the Mercy health system.

Could a Youngstown-Specific Model Perform Better?

Youngstown’s 12.1% prevalence (vs 7.5% system-wide) raises a natural question: would a dedicated market-level model yield stronger discrimination?

Potential advantages:
* Higher base rate makes the prediction task inherently easier — PR-AUC is base-rate-sensitive, so a model trained on a 12% positive population would be expected to report higher PR-AUC even at similar discrimination ability.
* A market-specific model could learn local referral patterns, provider behaviour, and population characteristics that are diluted in the system-wide model.

Practical concerns:
* Youngstown contributes only 24,281 patients (8% of cohort). After train/test split, the effective training set would be \~19K patients with \~2,350 positives — substantially less data for hyperparameter tuning and stable feature learning.
* Maintaining separate models per market multiplies operational complexity (retraining, monitoring, threshold calibration) without guaranteed net clinical benefit.
* The current all-market model already captures Youngstown’s higher-acuity signal implicitly through clinical features (utilisation patterns, comorbidity burden, lab values) that reflect the local disease environment.

Current position: The system-wide model remains the primary direction. Market-level performance breakdowns can be monitored post-deployment; if Youngstown shows materially different calibration or recall, a targeted adjustment (e.g. market-specific threshold) is a lighter-weight intervention than a fully separate model.

Why We Rebuilt the Model

What Prompted the Improvements

The earlier model showed that EHR-based prediction was feasible, but review of model behaviour identified an opportunity to make the model both more clinically understandable and more complete for complex patients.

The redesign focused on three goals:

  1. Translate features into clinical concepts rather than opaque database artifacts.
  2. Capture multi-source burden for patients whose diagnoses and treatment complexity appear across many parts of the chart.
  3. Retain or improve discrimination while reducing feature count and improving interpretability.

That redesign has now succeeded: the integrated model exceeds the earlier baseline while using a much smaller, clinically organised feature set.

Key Clinical Findings

Key Clinical Findings

  • The model now exceeds the original baseline. The leading integrated H2O model achieved test PR-AUC 0.387, slightly above the prior baseline of 0.382.
  • The signal is clinically coherent. The strongest contributors span prior utilization, diagnosis burden, treatment complexity, labs, and inpatient physiologic instability.
  • The model is more efficient. It uses about 74% fewer features than the older baseline approach while improving interpretability.
  • Hybrid integration added real value. Signals initially identified through gap analysis were validated and are now integrated directly into the core feature dimensions rather than maintained as a separate supplement.
  • The result is operationally flexible. Thresholds can be adjusted depending on how many patients care management is able to review.

Headline Interpretation

This means the model can flag about 10% of patients, with a 36% yield, 4.9× lift, and capture of 47% of future admissions.

This threshold is not fixed. If review capacity is lower, the threshold can be raised to produce a smaller, higher-yield list. If the goal is broader capture, it can be lowered to review more patients and identify more admissions.

Limitations and Recommendations

Limitations and Next Steps

Current Limitations

  • Some clinically relevant inputs still have incomplete capture, especially SDOH screening and portions of kidney-function data.
  • Blood pressure parsing and a small number of lab naming issues remain open data-quality items.
  • Validation so far is retrospective rather than prospective.

Current Project Position

  • Hybrid integration is complete.
  • Tier A enrichment is closed and remains historical context only.
  • The integrated all-11 design is now the current best model direction.

Next Steps

  1. Prospective validation on a newer cohort.
  2. Dashboard-based review workflow for clinical use.
  3. Threshold calibration based on real care-management capacity.
  4. Targeted data-quality fixes to strengthen future model refreshes.
